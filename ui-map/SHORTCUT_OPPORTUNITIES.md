# 단축키 전환 기회 조사 — 기존 op 대비 (2026-06-11)

> **목적**: `SHORTCUTS.md`(공식 단축키 일람)와 `scripts/hancom.js` 현재 구현을 전수 대조해,
> **단축키로 바꾸면 유리한 op**을 등급별로 정리. **조사만** — 구현/검증은 별도 편집 세션 몫.
> 현 구현 경로는 hancom.js 실코드 확인(2026-06-11, feat/mac-hancomdocs-compat + 미커밋 도형스타일 포함).

## 0. 전제 — "단축키 신뢰불가" 원칙의 재해석

MENU_MAP의 "webhwp 단축키 신뢰불가 → 클릭이 정석"은 **사람/headed 기준**(브라우저 단축키 우선 충돌, 기기별 차이 — 공식 도움말도 경고). 그러나 **Playwright 합성 키 이벤트는 브라우저 액셀러레이터를 안 탄다**(CDP로 렌더러에 직접 주입 — Ctrl+N이 새 창 안 열고, Ctrl+P가 인쇄 안 띄움). 즉 자동화 맥락에선 충돌 리스크의 대부분이 소멸.

**이미 우리 코드에 단축키 성공 사례 5종**(검증된 선례):
| 어디 | 키 | 비고 |
|---|---|---|
| format-text | `ControlOrMeta+B/I/U` | 진하게/기울임/밑줄 토글 — 안정 동작 중 |
| level | `Control+NumpadSubtract/Add` | **툴바 버튼이 오히려 inert**라 단축키가 유일 경로였음 |
| table-op merge/split | `F5`, `Shift+ArrowRight` | 셀 블록 |
| set-cell-text | `Home`, `Shift+End` | 줄 선택 |
| 공통 | `⌘+A`, `End`, `Enter`, `Escape`, `Tab` | 도처 |

- 키맵: headless Mac Chromium = `MacIntel` → **macOS 칼럼 적용**(⌘=Meta, ⌃=Control, ⌥=Alt).
- **미검증 영역 = 3키 시퀀스**(`⌃+N,T` 류: Ctrl+글쇠1 누르고 뗀 후 글쇠2). Playwright로는 `press('Control+KeyN')` → `press('KeyT')`. **1개만 PoC 통과하면(추천: ⌃+N,T 표 만들기) N/K계열 전부 신뢰 가능** — 최우선 검증 항목.

## 1. A급 — 즉시 큰 이득 (brittle한 다단 클릭/호버 체인 제거)

| op | 현재 경로 (실코드) | 단축키 대안 | 기대 이득 |
|---|---|---|---|
| **find / findText()** | 툴바 `a[title="찾기"]` 클릭 → 드롭다운 '찾기...' 가시항목 좌표 클릭 (2클릭+900ms 대기) | `⌘+F` 1키 | **findText는 모든 편집 op의 앵커 진입로 = 호출 빈도 최다.** 누적 시간·brittleness 절감 1위. 다이얼로그 이후 로직(검색칸/문서전체/다음찾기)은 그대로 재사용 |
| **footnote/endnote** | 입력 → '주석' 호버 → 서브메뉴 '각주'/'미주' (3단 호버 체인 — 메뉴 op 중 가장 brittle한 패턴) | `⌃+N,N` / `⌃+N,E` | 호버 체인 전부 제거. 3키 시퀀스 PoC 대상 |
| **object-prop 진입** | 객체 좌표 우클릭 → objMenuClick '개체 속성...' | 객체 **클릭 선택 후 `P`** 1키 | ① 우클릭 메뉴 탐색 제거 ② **선(line) 객체 한계 해소 후보**: 현재 "획이 얇아 우클릭 hit 거의 불가"가 문서화된 한계 — 클릭 선택만 되면 P로 진입 가능 |
| **object-prop 탭 직행** | 다이얼로그 열고 dlgClickText('채우기')/dlgClickText('선')으로 탭 클릭 | 객체 선택 후 **`C`**(채우기 탭 바로) / **`L`**(선 탭 바로) | 신규 --fill/--border 경로의 탭 탐색 단계 자체를 생략. 단 다이얼로그가 해당 탭으로 '열린' 상태로 시작하는지 확인 필요 |
| **hyperlink** | 입력 메뉴 → '하이퍼링크...' — **함정⑥: 메뉴 여는 동안 선택 풀림** → '표시할 텍스트' 명시 입력으로 우회 중 | `⌃+K,H` | 메뉴를 안 거치므로 **선택 보존 가능성** — 우회 로직 단순화 후보. 검증 가치 최상 |
| **table-op merge/split 확정** | F5 블록 후 표 메뉴 → '셀 합치기' / '셀 나누기...' 클릭 | 블록 상태에서 **`M`** / **`S`** 1키 | 메뉴 왕복 제거. S는 다이얼로그 열기까지(나누기 설정은 기존 다이얼로그 로직 재사용) |
| **page-break** | 쪽 메뉴 → `.p_page_break` 또는 '쪽 나누기' 텍스트 | `⌃+Return` 1키 | 메뉴 제거. ⚠️ **컨텍스트 충돌**: 표 안에서는 같은 키가 '줄 추가' — 캐럿이 표 밖인지 확인 필수 |

## 2. B급 — 다이얼로그/기능 직행 (메뉴 1~2클릭 절약, 위험 낮음)

| op | 현재 | 단축키 | 메모 |
|---|---|---|---|
| char-shape | 서식 메뉴 → `.char_shape` | `⌘+L` | 글자 모양 다이얼로그 직행. 이후 setDialogField 동일 |
| para-shape | 서식 메뉴 → `.para_shape` | `⌥+T` | 문단 모양 직행 |
| insert-table | 입력 메뉴 → `.insert_table` | `⌃+N,T` | **3키 시퀀스 PoC 1순위**(결과 다이얼로그가 명확해 판정 쉬움) |
| textbox | 입력 메뉴 경유 | `⌃+N,B` | |
| para-line | 입력 메뉴 → `.s_insert_line`/'문단 띠' | `⌃+N,L` | 다이얼로그 없는 즉시 액션이라 1키 전환 단순 |
| field(누름틀) | 입력 메뉴 → `.field`/'필드 입력...' | `⌃+K,E` | + '누름틀 고치기' `⌃+N,K`는 **누름틀 수정 op(미구현)** 거저 얻는 진입로 |
| caption | 입력 → '캡션 넣기' 호버 → 위치 서브메뉴 | `⌃+N,C` | ⚠️ **위치 선택이 단축키엔 없음**(기본 위치로 삽입 추정) — `--position` 지정 시엔 메뉴 유지, 기본값이면 단축키. 부분 대체 |
| table-op 줄/칸 | 표 메뉴 → '줄/칸 추가하기' 호버 → 방향 항목 (3단) | 표 안 커서: `⌃+Return` 줄추가 / `⌃+BackSpace` 줄지우기 / `⌃+I` 칸추가 / `⌃+D` 칸지우기 | ⚠️ **방향 제어 불가**(위쪽/왼쪽 추가 없음) — `insert-row-below`·`delete-*`만 대체 가능, `*-above`/`*-left`는 메뉴 유지. 추가 방향(아래? 어느쪽 칸?) 실측 필요 |
| 문자표 (미구현) | — | `⌘+F10` | 구현하게 되면 메뉴 대신 이걸로 시작 |
| 편집 용지 (미구현) | — | `F7` | 용지 방향/여백 op 만들 때 |

## 3. C급 — 보조·부분 대체 / 폴백 가치

| 대상 | 단축키 | 가치 |
|---|---|---|
| font-color 8색 fast-path | `⌃+M, K/R/B/D/G/Y/C/W` (검/빨/파/자주/초록/노랑/청록/흰) | 그 8색이면 팔레트 스와치 로직 생략. 그 외 색은 기존 팔레트 유지 |
| align | `⌃+⇧+ L/R/C/M/T` | 현재 툴바 클릭도 빠름 — 이득 미미. 단 **툴바 오버플로 숨김 함정① 폴백**으로 가치 |
| list 토글 | 글머리표 `⌃+⇧+BackSpace` / 문단번호 `⌃+⇧+I` | 현재 `.bullet_list`/`.number_list` 툴바 — 토글이라 상태 의존 동일. 폴백용 |
| z-order/배치 | 객체 선택 후 `⇧+PageUp/Down`(맨앞/맨뒤), `⇧+Home/End`(글앞/글뒤) | setObjectWrap front/behind 부분 대체 + **z-order op(미구현)** 거저 |
| 개체 미세 이동 | 객체 선택 후 방향키 | object-prop --pos(mm)의 보조 — 픽셀 nudge |
| 텍스트 블록 선택 | `F3×2` 낱말 / `F3×3` 문단 / `⇧+End` 줄끝 / `⌃+⇧+↑↓` 문단끝까지 | 현재 format 계열은 **마우스 드래그(픽셀 폭 계산) 선택** — 좌표 오차에 민감. 키보드 블록은 좌표 면역. "단락 전체 서식"류에 특히 |
| 셀 블록 계산 | `⌃+⇧+S/A/P` 합/평균/곱 | **미구현 op 후보** — 표 블록 잡고 1키면 끝 |
| undo/redo (op 없음) | `⌘+Z` / `⌘+⇧+Z` | **편집 실패 롤백 op** 신설 후보 — 사실상 공짜 |
| 배율 표준화 | `⌃+G,Q`(100%) / `⌃+G,P`(쪽맞춤) / `⌃+G,B`(폭맞춤) | 캡처 전 배율 리셋 — capture 일관성 보조 |
| 조판/문단 부호 | `⌃+G,C` / `⌃+G,T` | 디버그 가시화 |

## 4. 단축키로 대체 **불가** (현행 유지)

- **bookmark · memo · equation · insert-chart · shape(도형) · 그림(파일선택)** — 공식 단축키 없음(그림 ⌃+N,I는 있으나 파일 업로드 흐름은 동일).
- **취소선(strike)** — 단축키 없음(B/I/U만). 현행 툴바 유지.
- **절대값 설정 계열** — 자간/장평/크기/줄간격의 단축키는 전부 **증감**(넓게/좁게/크게/작게)이라 절대값 지정인 우리 op과 의미 불일치. 다이얼로그 유지. (증감형 옵션을 추가한다면 활용 가능)

## 5. 공통 리스크 (편집 세션이 검증 시 유의)

1. **3키 시퀀스 미검증** — `⌃+N,T` 1개 PoC 먼저. 실패하면 A/B급 절반이 무효.
2. **단일 키(P/L/C/M/S/H/W)는 선택 상태 전제** — 선택이 풀린 채 누르면 **본문에 그 글자가 입력됨**(문서 오염!). 적용 후 다이얼로그 출현 확인 + 실패 시 즉시 `⌘+Z` 가드 필수.
3. **컨텍스트 충돌**: `⌃+Return` = 표 밖 쪽나누기 / 표 안 줄추가. `⌃+I` = 표 안 칸추가 / (서식 충돌 가능성). 캐럿 컨텍스트 확인 후 발사.
4. **macOS Fn 의존**: F7 등 기능키가 headless에서 그대로 가는지 — Playwright는 OS 키맵 안 타므로 괜찮을 것으로 추정되나 F7(편집용지)로 확인.
5. 전환하더라도 **기존 메뉴 경로는 폴백으로 보존** 권장(단축키 → 실패 감지 → 메뉴 폴백). level처럼 역방향 사례(툴바 inert, 단축키만 동작)도 있으므로 op별 실측이 정답.

## 6. 추천 착수 순서 (편집 세션용)

1. **PoC: `⌃+N,T`** (3키 시퀀스 동작 확인 — 이 결과가 전체 판도 결정)
2. **find `⌘+F`** (빈도 최다 — 전 op 공통 이득)
3. **footnote/endnote `⌃+N,N/E`** (가장 brittle한 호버 체인 제거)
4. **object-prop `P`/`L`/`C`** (선 객체 한계 해소 + 신규 fill/border 탭 직행)
5. **hyperlink `⌃+K,H`** (선택 보존 검증)
6. merge/split `M`/`S` → page-break `⌃+Return` → B급 일괄
